# Release Notes: sqlh v0.8.0

**Release date**: 2026-06-12
**Compare**: https://github.com/kirill-scherba/sqlh/compare/v0.7.1...v0.8.0

## Highlights

### Type-safe WHERE Helpers
No more raw SQL strings for WHERE conditions. Every standard operator now has a fluent, chainable constructor:

| Operator | Helper | Example |
|----------|--------|---------|
| `=` | `sqlh.Eq` | `sqlh.Eq("name", "foo")` |
| `<>` | `sqlh.Ne` | `sqlh.Ne("status", "deleted")` |
| `>` | `sqlh.Gt` | `sqlh.Gt("age", 18)` |
| `>=` | `sqlh.Gte` | `sqlh.Gte("score", 100)` |
| `<` | `sqlh.Lt` | `sqlh.Lt("age", 65)` |
| `<=` | `sqlh.Lte` | `sqlh.Lte("price", 99.99)` |
| `LIKE` | `sqlh.Like` | `sqlh.Like("email", "%@example.com")` |
| `IN` | `sqlh.In` | `sqlh.In("id", 1, 2, 3)` |
| `IS NULL` | `sqlh.IsNull` | `sqlh.IsNull("deleted_at")` |
| `IS NOT NULL` | `sqlh.IsNotNull` | `sqlh.IsNotNull("confirmed_at")` |

These helpers prevent typo-prone string concatenation and make intent explicit at the call site.

### Performance Benchmarks
A new `bench/` module provides head-to-head performance comparisons across four approaches:

| Approach | Insert | Get | List | Update | Delete |
|----------|--------|-----|------|--------|--------|
| raw `database/sql` | ✅ | ✅ | ✅ | ✅ | ✅ |
| `sqlx` | ✅ | ✅ | ✅ | ✅ | ✅ |
| `sqlh` | ✅ | ✅ | ✅ | ✅ | ✅ |
| GORM | ✅ | ✅ | ✅ | ✅ | ✅ |

24 benchmark functions cover all CRUD operations so you can make an informed choice.

### Code Comparison Examples
`examples/comparison/` shows the same CRUD workflow written in raw `database/sql`, `sqlx`, and `sqlh` — side by side. The difference in lines of code is dramatic (typically 60–80 % less boilerplate with `sqlh`).

### Animated GIF Demo
A zero-boilerplate CRUD walkthrough is now available as an animated terminal recording in the README (`docs/demo.gif`).

### Documentation Improvements
- **pkg.go.dev badge** replaces the deprecated `godoc.org` link.
- **`Table[T]` wrapper API examples** are now rendered on pkg.go.dev via `Example_` functions.
- Backward compatibility restored for exported `query.Update` and `query.Delete` query generators.
